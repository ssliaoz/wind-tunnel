系统概述
智慧风洞管理系统是一个集成化的管理平台，旨在对风洞试验的全生命周期、设备资产、安全风险、知识库及系统基础进行数字化管理。系统核心功能围绕试验流程、设备运维与数据分析展开。

一、试验管理
1.1 试验申请
实验室管理
功能描述: 创建实验室信息。
备注: 实验创建表单无需过于详细的参数填写，技术参数可作为备注。
试验项目创建
功能描述: 针对风洞试验立项场景，通过系统标准化表单录入试验名称、核心目的、计划周期、负责人、参与团队等信息，支持试验类型（空气动力学 / 结构强度 / 气动噪声等）分类标注与必填项校验，生成唯一试验项目编号，为后续审批、排期奠定数据基础。
备注: 包含工况名称、备注，支持多个试验关联，组合试验（单个试验进行串联）。
试验审批流转
功能描述: 针对试验项目合规性审核需求，构建 “申请人提交→部门负责人审核→终审归档” 多节点在线审批流程，支持审批意见实时批注、附件上传（如技术方案）、驳回原因明确反馈，系统自动跟踪审批进度并推送节点提醒。
备注: 审批环节需要确认。
试验排期管理
功能描述: 针对多设备 / 多试验冲突规避需求，提供可视化日历排期界面，支持按试验室、设备类型筛选可用时段，自动关联设备运行状态（空闲 / 占用 / 维护），排期后生成试验日程表并同步至相关人员账号，支持排期调整与冲突预警。
备注: 日历适配中国日历。国际化需求暂不开发。
1.2 试验数据采集
实时数据采集
功能描述: 针对风洞试验数据实时监测需求，通过 CWT1 PC、CWT2 PC、CWT3 PC、AAWT PC、公共动力系统PC接口，获取系统数据内容。
备注:
接口协议为TCP/IP。
接口清单和字段说明书后续提供。
一台电脑（公共IP）有多个端口，对应多个接口，数据格式统一。
按设备进行可视化数据展示（显示最新数据），而非跟随某个实验。可视化页面应美观大方。
数据库及存储：实时数据采用MongoDB存储，历史数据保留周期可参考10年。实验管理数据使用MySQL/MongoDB混合模式。
手动数据补录
功能描述: 针对传感器故障、特殊工况等场景，提供手动数据录入界面，支持CWT1 PC、CWT2 PC、CWT3PC、AAWT PC、公共动力系统PC接口数据手动补录，记录补录人、补录原因及时间戳。
备注:
功能权限根据菜单管控。
数据权限管理初步规划需支持到实验室级别，不同角色拥有不同操作与查看权限。目前有四个实验室。
1.3 数据分析中心
试验室维度
功能描述: 针对试验室运营成本优化需求，按单个 / 多个试验室维度，统计指定周期内试验总次数、总能耗、人工成本、耗材消耗等数据，生成成本对比报表，支持筛选试验类型、时间范围，为试验室资源调配提供数据支撑。
备注:
不确定CWT1 PC等接口是否包含人工成本、耗材消耗数据。
人工成本、耗材消耗等数据可通过后台配置一次性录入，后续生成报告时参考使用。
客户关注试验台的利用率、故障率、使用率。
设备维度
功能描述: 针对设备能耗管控需求，聚焦单台 / 多台核心设备（风机 / 风洞本体 / 控制系统），统计指定周期内运行时长、电 / 水 / 气能耗分项数据、维护损耗成本，生成设备成本排行与趋势曲线，识别高能耗设备。
能耗维度
功能描述: 针对能耗成本精细化管理需求，按电、水、气等能耗类型，拆分各试验室 / 各试验项目的能耗用量、单价、总成本，支持日 / 周 / 月 / 年周期统计，生成能耗成本占比饼图，为能耗优化方案制定提供依据。
试验维度
功能描述: 针对单试验 / 同类试验成本核算需求，拆解试验全流程成本（人工薪酬 + 设备折旧 + 能耗 + 耗材 + 维护），支持同类试验成本横向对比（如不同批次空气动力学试验），生成成本构成明细报表。
时间维度
功能描述: 针对成本趋势监控需求，按日 / 周 / 月 / 年周期统计各试验室 / 全系统成本变化趋势，支持设置成本基准值，自动标注超支时段，生成趋势曲线与异常分析备注。
大屏展示
功能描述: 展示当前试验数据。新建子页面，适配显示数据信息。提出需要为运维人员提供一块独立的大屏幕用于实时数据监控。
数据可视化展示
功能描述: 针对数据直观呈现需求，将各类分析结果转化为热力图（试验室成本分布）、趋势曲线（能耗变化）、饼图（成本构成）、柱状图（设备能耗排行）等可视化图表，支持图表放大、导出（PDF）、打印。
1.4 成本管控
成本核算与预警
功能描述: 针对成本超支风险防控需求，支持按试验室、设备、试验项目设定成本阈值（绝对金额 / 相对占比），系统实时核算实际成本与阈值差额，超支时自动触发短信 / 系统弹窗提醒，同步推送超支明细。
备注: 试验中的成本信息是否可以通过接口获取？需要通过手工补录？暂不开发，需与客户进一步确认。
1.5 试验报告管理
报告自动生成
功能描述: 针对试验报告标准化输出需求，系统基于采集的试验数据、分析图表、试验参数自动生成标准化报告，内置风洞行业专用模板（含试验目的、设备参数、数据记录、分析结论等模块），支持自动填充数据与图表，减少手动编写工作量，生成PDF文件信息。
备注: 具体生成内容需客户提供现成模板参考。获取的信息不全，对客户的实际作用不大。暂不开发，需与客户进一步确认。
报告归档与检索
功能描述: 针对报告高效管理需求，按试验类型、时间、负责人、项目编号等维度对报告进行分类存储，支持关键词检索（如 “202508 + 风机 + 气动噪声”）、多条件组合筛选，报告关联对应试验项目数据，支持在线预览、下载。
备注: 获取的信息不全，对客户的实际作用不大。暂不开发，需与客户进一步确认。
二、设备管理
2.1 基础信息管理
设备静态数据录入
功能描述: 针对设备全生命周期管理基础需求，通过标准化表单录入风洞设备核心信息（名称、型号、KKS 编码、技术参数、生产厂商、安装位置、采购日期、质保期、校准周期），支持上传设备图纸、说明书等附件，生成唯一设备档案，为后续维护、监控提供数据支撑。
备注: 包括试验台的基础信息。需明确设备归属的层级。设备信息的基础静态数据管理和定期的维护保养计划是核心需求。
设备分类管理
功能描述: 针对设备高效管理需求，按风洞设备功能属性分类（核心设备：风机 / 风洞本体 / 传感器 / 控制系统；辅助设备：真空泵 / 冷却系统 / 供电设备），支持自定义分类标签，按分类筛选设备列表。
备注: 设备种类很多，有大类和小类的信息。客户可自行创建分类和设备。
2.2 维护计划管理
预防性维护计划
功能描述: 针对设备预防性保养需求，按设备类型、运行时长、行业规范设定维护周期（如风机每运行 500 小时保养），系统自动生成维护任务（含维护项目、标准流程、所需备件 / 工具），支持维护计划调整、暂停。
备注: 定期通知，系统提醒。功能需明确区分内部维护（如日检、周检）和外部维护（如年检）。外部维护需提前推送通知。
维护任务派发
功能描述: 针对维护任务高效执行需求，系统将生成的维护任务推送至对应维护人员账号（系统通知），维护人员可接收任务、反馈进度（未开始 / 进行中 / 已完成），管理员实时查看任务执行状态。
备注: 初步确定固定人员维护特定设备或子系统。关于“指派人”的任务派发机制，需与客户进一步确认。
2.3 维修管理
维修记录
功能描述: 针对设备故障快速响应需求，支持在线提交故障报修单（填写故障设备编号、故障描述、故障位置、现场图片、紧急程度、维修报告），选择分配至对应维护人员，支持报修进度跟踪。
备注: 将保留对巡检、更换等事件的记录，简化流程，无需在线上报修单。
维修记录归档
功能描述: 针对维修历史追溯需求，将所有维修记录（故障信息、维修方案、效果评估）按设备编号、故障类型分类归档，支持关键词检索（如 “风机 + 轴承故障”），维修记录关联设备台账，为设备健康分析提供数据支撑。
2.4 健康状态监测
故障实时预警
功能描述: 针对设备故障早发现需求，基于 CWT1 PC、CWT2 PC、CWT3 PC、AAWT PC、公共动力系统PC接口数据，识别故障，通过站内信 / 短信 /邮件 系统弹窗多渠道推送预警信息（含设备编号、异常参数、风险等级）。
备注: 实现困难：需要通过大数据量来学习。旨在提前发现问题，但技术实现较复杂，需后续研究。
故障报警
功能描述: 针对设备故障早发现需求，基于 CWT1 PC、CWT2 PC、CWT3 PC、AAWT PC、公共动力系统PC接口数据，识别故障，通过站内信 / 短信 /邮件 系统弹窗多渠道推送预警信息（含设备编号、异常参数、风险等级）。
备注: 需要提供接口中的异常码与风险等级的映射关系。
2.5 校准计划制定
校准周期设定
功能描述: 针对设备计量精度保障需求，按国家计量规范、设备厂商要求设定设备校准周期（如传感器每 6 个月校准 1 次），系统自动生成校准任务（含校准项目、标准值、校准机构），支持校准计划调整。
备注: 已接收到第一条数据的时间为开始计算校准周期的起始时间。涉及温度计量、风速等。
2.6 校准记录管理
校准报告存储
功能描述: 针对校准历史追溯需求，存储设备校准报告（含校准数据、误差值、是否合格），记录校准时间、校准机构、校准人员，支持不合格设备整改记录上传，校准记录关联设备台账。
三、安全管理
3.1 风险台账管理
风险点录入
功能描述: 针对安全风险源头管控需求，登记风洞全场景风险点（如高压触电、设备倾覆、气体泄漏、火灾），标注风险等级（一般 / 较重 / 严重）、影响范围、防控措施，生成风险台账。
备注: 功能保留，但风险点的录入、排查等方式需要与客户确认具体需求（例如是否整合进常规巡检）。
隐患排查记录
功能描述: 针对隐患闭环管理需求，记录定期隐患排查结果（排查人员、时间、区域、隐患描述），制定整改措施、明确责任人与完成时限，跟踪整改进度，形成 “排查 - 整改 - 复查” 闭环记录。
四、知识库
4.1 知识录入与维护
知识上传/更新
功能描述: 支持拓展名:.txt，.csv，.md，.json，.doc，.docx，.pdf，单个文件大小在200MB以内。支持知识信息更新。
备注: 将采用基于客户自行录入案例的本地数据库（MySQL）知识库方案，而非依赖大模型或知识图谱。
知识查询
功能描述: 根据上传的标题、知识类别、时间来查询知识库信息。
五、系统管理
5.1 用户管理
组织架构管理
功能描述: 创建系统的组织架构信息。
备注: 是否需要根据现有系统的组织架构信息进行同步？手工录入。
用户信息维护
功能描述: 针对系统用户规范化管理需求，支持新增 / 删除 / 修改用户信息（姓名、账号、密码、部门、角色），设置用户状态（启用 / 禁用），记录用户创建时间。
备注: 是否需要根据现有系统的用户信息进行同步？手工录入。 用户并发数量按照 100人设计。
5.2 权限配置
角色权限分配
功能描述: 针对系统数据安全需求，按角色（管理员 / 试验员 / 维护员 / 访客）分配菜单操作权限（查看 / 编辑 / 删除）、数据查看权限（全量数据 / 指定试验室数据），支持自定义角色与权限组合。
备注: 后台进行手动录入和权限分配。
5.3 站内信
站内信
功能描述: 为系统用户提供站内信的接受和发送功能。
5.4 登录/退出
NULL
功能描述: 提供用户登录和退出系统。
备注: 单点登录功能，因客户无统一身份认证系统，本次暂不开发。
5.5 基础参数配置
系统参数设置
功能描述: 针对系统个性化适配需求，设置试验数据单位（如风速 m/s/km/h）、预警阈值（如能耗超支比例）、审批流程节点、数据采集频率等基础参数，支持参数修改与生效预览。
5.6 通知配置
通知方式设置
功能描述: 针对系统提醒精准触达需求，配置预警通知方式（短信 / 邮箱 / 系统弹窗）、接收人分组（按角色 / 部门），设置通知发送频率（即时 / 定时汇总）。
5.7 操作日志
操作行为记录
功能描述: 针对系统安全追溯需求，记录所有用户的登录 / 退出、菜单操作、数据修改、文件上传 / 下载等行为，包含操作人、时间、IP 地址、操作内容，支持按用户 / 时间 / 操作类型检索。
5.8 系统日志
系统运行记录
功能描述: 针对系统运维支撑需求，记录系统运行状态（启动 / 停止）、故障信息（报错代码、原因）、数据同步情况、备份记录，支持按时间 / 日志类型检索，为系统故障排查提供依据。
